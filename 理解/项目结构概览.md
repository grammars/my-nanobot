# Nanobot 项目结构概览

## 项目简介

Nanobot 是一个开源、超轻量级的 AI 代理框架，灵感来源于 OpenClaw、Claude Code 和 Codex。它保持核心代理循环小巧且可读，同时支持聊天渠道、记忆系统、MCP 和实用的部署路径，让您可以从本地设置快速过渡到长期运行的个人代理，开销极小。

## 项目版本信息

- **版本**: v0.1.5.post2
- **Python 要求**: ≥3.11
- **许可证**: MIT

## 核心特性

- 多渠道聊天支持（Telegram、Discord、Slack、飞书、钉钉、微信等）
- 记忆系统（包括 Dream 记忆整合）
- MCP（Model Context Protocol）支持
- 多种 LLM 提供商集成（OpenAI、Anthropic、Azure OpenAI、GitHub Copilot 等）
- 定时任务和心跳服务
- Web UI 界面
- Docker 容器化部署

## 主要模块详解

### 1. nanobot/ - 主项目目录

这是项目的核心目录，包含所有 Python 源代码。

代理系统是项目的核心，负责处理用户输入、调用工具、生成响应。

**主要文件:**
- `loop.py` - 代理循环引擎，核心处理逻辑
- `runner.py` - 代理运行器
- `memory.py` - 记忆系统（MemoryStore、Consolidator、Dream）

**tools/ 子目录** - 工具系统:

## 核心工作流程

1. **用户输入** → 通过 CLI、API 或聊天渠道接收
2. **消息处理** → 消息总线路由到代理循环
3. **代理循环** → 处理输入、调用工具、生成响应
4. **工具执行** → 文件操作、搜索、Web 请求等
5. **响应生成** → LLM 生成回复
6. **输出发送** → 通过相应渠道返回给用户

## 部署方式

1. **本地开发**: 直接运行 Python 脚本
2. **Docker 部署**: 使用 docker-compose 快速部署
3. **多实例部署**: 支持多个代理实例运行

## 总结

Nanobot 是一个功能丰富、架构清晰的 AI 代理框架。它通过模块化设计，将核心代理逻辑、工具系统、渠道集成、记忆管理等功能分离，便于扩展和维护。项目支持多种部署方式和聊天平台，适合构建个人 AI 助手或企业级聊天机器人应用。
